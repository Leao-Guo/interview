面试官最看重什么？

  • 基础扎实：JVM、集合、并发、Spring 原理要说出“为什么”。
  • 项目深度：能量化业务结果，讲清你的真实贡献和取舍。
  • 问题定位：遇到线上故障时的排查路径、监控指标与复盘能力。
  • 沟通结构：先结论、再原理、最后案例，避免散点式表达。

Java 高频面试题（含回答要点）

1) HashMap 为什么线程不安全？

回答时要提到并发扩容、链表/树结构写入竞争与数据覆盖问题，并补充 ConcurrentHashMap 的分段/桶级并发思想。

2) JVM 内存模型与 GC 如何影响性能？

重点描述堆分代、对象晋升、常见 GC 算法与停顿时间（STW）对接口 RT 的影响，再结合调优案例更有说服力。

3) 说一下 Spring AOP 与事务失效场景

必须点出代理机制（JDK/CGLIB）、自调用失效、异常类型与传播行为，展示你对框架“边界条件”的理解。

4) MySQL 索引什么时候会失效？

围绕最左前缀、函数操作、隐式类型转换、范围查询后列失效来讲，并给出 EXPLAIN 分析过程。

5) Redis 缓存击穿/穿透/雪崩如何治理？

分别给出互斥锁、布隆过滤器、随机过期时间与多级缓存方案，强调高并发下的降级与限流策略。

6) 线程池参数怎么设定？

结合 CPU 密集/IO 密集模型说明 corePoolSize 与 queue 策略，能讲拒绝策略与监控报警就是加分项。

提高面试通过率的可执行方法

构建“八股 + 项目”映射：每个知识点都绑定一个项目场景，回答更真实。
准备 STAR 案例：至少 3 个难点问题，覆盖性能优化、稳定性、协作推进。
输出标准答题模板：结论 → 原理 → 方案 → 结果，控制在 1~2 分钟。
模拟面试复盘：录音回放，修正口头禅、逻辑跳跃与术语不准确问题。
针对 JD 定制：按岗位关键词准备“必答清单”，减少无效复习。
避坑提醒：不要背诵式回答，遇到不会的问题要“先拆解再给方案”。
建议：面试前 24 小时做一次“项目深挖 + 高频快问快答”冲刺，状态会明显提升。
